# portal-wd-archive

本仓库用于归档与发布多个独立的 HTML 页面。这些页面可能包含各类信息展示、工具界面、静态站点或原型演示。仓库本身不针对任何单一域名或具体网站，仅作为 HTML 资源的集合与发布入口。

## 项目简介

portal-wd-archive 是一个轻量级的 HTML 页面归档仓库。所有页面均为独立、自包含的 HTML 文件，可直接通过浏览器打开查看。适合存放临时页面、离线资料、演示文稿或小型项目的前端界面。

本仓库不依赖后端服务，所有内容均为静态资源，方便快速分享与存档。

## 目录说明

```
portal-wd-archive/
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 仓库入口页面（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录或独立文件对应一个归档页面。
- `assets/`：存放多个页面可能共用的样式、脚本或媒体文件。
- `index.html`：可选的导航或索引页面，方便浏览所有归档内容。

## 页面归档说明

- 每个页面均为独立 HTML 文件，拥有完整的文档结构。
- 页面之间无强制依赖关系，可单独访问。
- 部分页面可能引用 `assets/` 下的公共资源，请确保路径正确。
- 归档内容可能包含静态数据、界面原型、信息卡片或实验性页面。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并按需更新 `index.html` 中的链接。
- 资源更新：修改 `assets/` 下的文件时，请注意检查所有引用页面的兼容性。
- 删除页面：移除对应文件及 `index.html` 中的引用。
- 建议保持目录结构清晰，避免文件混乱。

## 使用方式

1. 克隆仓库到本地。
2. 直接打开 `pages/` 下的任意 HTML 文件，或通过 `index.html` 浏览页面列表。
3. 也可通过 GitHub Pages 或其他静态托管服务发布本仓库内容。
